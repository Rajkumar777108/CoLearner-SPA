What is Abstract Class/ All about abstract class?

An abstract class is a special kind of class that has no implementation. It cannot be instantiated. Its implementation logic is provided by the classes that derive from it. It can have both abstract as well as non-abstract methods. It is not compulsory to have only abstract methods in an abstract class. We can also have an abstract class with only non-abstract methods.

  • An abstract class cannot support multiple inheritances. If we try to inherit more than one abstract class then it will throw compile time error as below.

Key Point: -

  • We cannot create an object of Abstract class, but we can create a reference of it. And assign class’s object to this reference and call abstract class’s method like below. 

        public abstract class ABS1
        {
            public void NonAbstractMedhod()
            {
                Console.WriteLine("Non-Abstract Method");
            }  
            public abstract void ABSMeth1();           
        }  
        public class CLS1 : ABS1
        {
            public override void ABSMeth1()
            {
                Console.WriteLine("ABSMeth1");
            }  
        }
        class Program 
        {
            static void Main(string[] args)
            {
                CLS1 obj = new CLS1()
                ABS1 obj1; //Reference created of Abstract class. 
                obj1 = obj;//Assign class's object to Abstract class reference variable. 
                obj1.ABSMeth1();//Call abstract method with reference variable.
                obj1.NoNAbstractMedhod();
                Console.ReadLine()
            }

  • An inheritance between abstract to abstract class is possible , we don't need to implement abstract method of base class into derived class.We can implement it later in concrete class. 

        public abstract class ABSCls
        {
            public abstract void ShowMessage();
        }
        public abstract class ABSCls1 : ABSCls
        {
            public void NonAbsMethod()
            {
                Console.WriteLine("NonABS funtion");
            }
            //No need to provide implememtation of base class abstract function, we can implement it in below concrete class directly 
        }
        public class ConcreteCls : ABSCls1
        {
            public override void ShowMessage()
            { 
                 Console.WriteLine("Abstract function implementation");
            }
        }

        class Program 
        {
            static void Main(string[] args)
            {
                ConcreteCls obj = new ConcreteCls()  
                obj.ShowMessage();
                obj.NonAbsMethod();
                Console.ReadLine()
            }
        }

  • It's mandatory to implement all abstract function of base class into derived class.

  • we cann't create abstract methods inside non-abstract class but an abstract can have non-abstract function.An abstract class can exists without any abstract function(an abstract class with only non-abstract function is possible).

  • An abstract class can never be static or sealed, if you try to use then it weill throw compile time error.
